# Data-Modelling-Governance-for-Customer-Risk-Assessment-Banking-Use-Case-

## Overview
This project demonstrates how business requirements in a banking environment can be translated into data requirements, data flows, data models, and governance controls.

The use case focuses on customer risk assessment using multiple data sources such as customer profile, ownership structure, geographical risk, transaction behaviour, customer business activity, and UBO information.

## Objective
- Translate business requirements into data requirements
- Design a simple data model
- Define data flows across systems
- Apply governance and compliance controls
- Show how structured data supports customer risk assessment

## Project Structure
- Business Context
- Data Flow
- Data Model
- Governance Framework
- Use Case Walkthrough

## Why This Matters
Customer risk assessment in banking cannot rely on a single data point. It requires combining information from multiple sources and structuring it in a clear, controlled, and explainable way to support reliable decisions.
